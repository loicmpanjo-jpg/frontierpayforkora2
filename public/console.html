Environment Configuration

# KORA
KORA_BASE_URL=https://api.korapay.com/merchant
KORA_PUBLIC_KEY=YOUR_KORA_PUBLIC_KEY
KORA_SECRET_KEY=YOUR_KORA_SECRET_KEY

# VISA
VISA_BASE_URL=https://sandbox.api.visa.com
VISA_KEY_ID=YOUR_VISA_KEY_ID
VISA_API_KEY=YOUR_VISA_API_KEY
VISA_CERT_PATH=./certs/cert.pem
VISA_KEY_PATH=./certs/private.key

⚠️ Security Notice

Never place real API secrets in frontend HTML or public repositories. Use backend-only .env configuration with proper .gitignore protection.

Kora API Integration

Kora Adapter Configuration

require('dotenv').config();

const axios = require('axios');

const koraClient = axios.create({
  baseURL: process.env.KORA_BASE_URL,
  headers: {
    Authorization: `Bearer ${process.env.KORA_SECRET_KEY}`,
    'Content-Type': 'application/json'
  },
  timeout: 30000
});

module.exports = koraClient;

Visa Sandbox Integration

Visa mTLS HTTPS Agent

const https = require('https');
const fs = require('fs');

const visaAgent = new https.Agent({
  cert: fs.readFileSync(process.env.VISA_CERT_PATH),
  key: fs.readFileSync(process.env.VISA_KEY_PATH),
  rejectUnauthorized: true
});

Security Best Practices

❌ Never Do

  • ✗ Expose secret keys in frontend
  • ✗ Commit .env to GitHub
  • ✗ Store Visa certificates publicly
  • ✗ Hardcode API secrets in HTML
  • ✗ Log sensitive data to console
  • ✗ Use innerHTML with user input

✅ Always Do

  • ✓ Use environment variables
  • ✓ Add .env to .gitignore
  • ✓ Store certs in .gitignore
  • ✓ Verify webhook signatures
  • ✓ Use HTTPS in production
  • ✓ Validate all inputs

Security Headers Implemented

X-Content-Type-Options: nosniff
X-Frame-Options: DENY
X-XSS-Protection: 1; mode=block
Content-Security-Policy: default-src 'self'
